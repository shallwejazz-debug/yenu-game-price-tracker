# 🤖 자동화 수집 아키텍처 설계 (무인 가격 수집)

> 목표: **100개 이상의 게임을 사람 검수 없이 자동으로 가격 수집·갱신**
> 핵심 원칙: "관리자는 게임 제목만 던져두면, 나머지는 시스템이 알아서 한다."

---

## 1. 왜 자동화가 필요한가 (문제 정의)

현재 구조(수동)의 한계:
- 관리자가 게임마다 → 에디션(PC/PS5/PS4...)마다 → 검색어/키워드를 직접 입력
- 100개 게임 × 평균 3~4 플랫폼 = **300~400개 에디션을 손으로 등록·검수**해야 함
- 가격은 시시각각 변하는데, 매번 사람이 "가격 수집" 버튼을 누를 수 없음

→ **결론: 등록도 자동, 수집도 자동, 분류도 자동이어야 함.**

---

## 2. 이미 검증된 핵심 기술 (이게 자동화의 근거)

### ✅ 제목만으로 플랫폼 자동 분류가 가능하다 (실측 완료)

`"엘든링"` 한 단어로만 네이버 검색 → 결과 30건의 **상품 제목**을 분석:

```
"엘든링" 단순 검색 결과
 → [PC]    3건   ("엘든링 스팀", "ELDEN RING PC" ...)
 → [PS5]  22건   ("엘든링 PS5 플레이스테이션5" ...)
 → [PS4]   3건   ("엘든링 PS4" ...)
 → [XBOX]  2건   ("엘든링 엑스박스" ...)
```

상품 제목 안에 거의 항상 **플랫폼 키워드(PS5/PS4/XBOX/SWITCH/스팀/PC)**가 들어있음.
→ 사람이 에디션을 미리 나눌 필요 없이, **수집 후 제목을 보고 자동으로 플랫폼별로 분류** 가능.

이게 자동화의 핵심 발견입니다. 이 위에 전체 시스템을 설계합니다.

---

## 3. 전체 아키텍처 (한눈에)

```
┌─────────────────────────────────────────────────────────────┐
│  관리자가 하는 일 (딱 2가지)                                  │
│   1) 게임 제목만 추가   예) "젤다의 전설 왕국의 눈물"         │
│   2) 쇼핑몰별 레퍼럴 ID 한 번 입력 (쿠팡/G마켓/11번가)        │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  ① 수집 잡 (Cron Trigger, 무인 자동 실행)                    │
│     - 등록된 모든 게임을 순회                                 │
│     - 네이버 쇼핑 API로 제목 검색 (display=40, sort=sim)      │
└──────────────────────────┬──────────────────────────────────┘
                           │ 원시 상품 리스트
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  ② 자동 분류 파이프라인 (사람 검수 없음)                     │
│     1. 게임 상품인가?     → category3="게임타이틀" 필터       │
│     2. 굿즈/중고 거르기   → 금지어 필터                       │
│     3. 가격 합리적인가?   → 5,000~300,000원 범위             │
│     4. 어느 플랫폼인가?   → 제목에서 PS5/PS4/PC... 자동 감지  │
│     5. 디지털 vs 패키지?  → 키워드 + 쇼핑몰로 판단           │
│     6. 어느 에디션?       → 없으면 자동 생성(upsert)         │
└──────────────────────────┬──────────────────────────────────┘
                           │ 정제된 (에디션, 가격) 데이터
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  ③ 저장 (D1)                                                 │
│     - 에디션 자동 upsert (game_id + platform 기준)           │
│     - 가격 insert → price_history 역대최저가 자동 갱신       │
│     - 신뢰도 점수(confidence) 같이 저장                       │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  ④ 검수 큐 (선택적, 사람은 "이상한 것만" 본다)               │
│     - confidence 낮은 항목만 관리자 대시보드에 표시          │
│     - 정상 항목은 자동 노출, 사람 손 안 탐                    │
└─────────────────────────────────────────────────────────────┘
```

**핵심:** 사람은 ④의 "이상한 것"만 가끔 본다. 정상 99%는 무인 처리.

## 4. 자동 분류 파이프라인 상세 (가장 중요)

수집한 상품 1건이 들어오면 아래 순서로 통과/탈락을 판정합니다.

### Step 1. 게임 상품 필터 (이미 구현됨)
- `category3 === "게임타이틀"` 인 것만 통과
- 콘솔 본체, 컨트롤러, 기프트카드 자동 제외

### Step 2. 금지어 필터 (이미 구현됨 + 중고 추가)
```
굿즈류: 굿즈, 피규어, 커버, 스티커, 키링, 포스터, 인형, 쿠션, 케이스
중고류: 중고, 중고품, 리퍼, 개봉  ← 신규 추가 권장
```

### Step 3. 가격 범위 필터 (이미 구현됨)
- 5,000원 ~ 300,000원 → 비정상 가격(1원, 가격문의 등) 제거

### Step 4. 플랫폼 자동 감지 ⭐ (자동화 핵심)
상품 제목을 소문자/공백정규화 후 키워드 매칭:

```
PS5    : "ps5", "플레이스테이션5", "플스5"
PS4    : "ps4", "플레이스테이션4", "플스4"
XBOX   : "xbox", "엑스박스", "엑박", "series x", "series s"
SWITCH : "switch", "닌텐도 스위치", "스위치", "ns "
PC     : "스팀", "steam", "pc ", "(pc)", "에픽", "epic"
ETC    : 위 어디에도 안 걸리면 ETC로 보류 → 검수 큐로
```

> 우선순위 주의: "PS5"가 "PS4"보다 먼저 매칭되도록, 또 "스위치"가
> 다른 단어에 섞이지 않도록 경계(공백/괄호)를 함께 검사.

### Step 5. 디지털 / 패키지 판정
- 네이버 쇼핑(쿠팡/G마켓/11번가)은 거의 **패키지(실물)** → `is_digital = 0`
- 디지털(스팀/PSN/닌텐도 e숍)은 네이버에 잘 안 잡힘 → **별도 수집 소스**로 처리
  - 1차: 네이버 결과는 전부 패키지로 저장 (현재 동작)
  - 2차(추후): Steam/PSN 가격은 별도 어댑터로 수집

### Step 6. 에디션 자동 생성 (upsert)
- `(game_id, platform)` 조합이 없으면 → 에디션 자동 생성
- 있으면 → 그 에디션에 가격만 추가
- 즉, **관리자가 에디션을 미리 안 만들어도 됨**

### 신뢰도(confidence) 점수 부여
각 상품에 0~100점:
```
+40  플랫폼이 명확히 감지됨 (ETC 아님)
+30  게임 제목 핵심 단어가 상품 제목에 포함됨
+20  가격이 같은 에디션 다른 상품들의 중앙값 ±40% 이내
+10  신뢰 쇼핑몰(쿠팡/G마켓/11번가/네이버)
-----
점수 < 50  → 검수 큐로 (사람이 본다)
점수 >= 50 → 자동 노출
```

## 5. 무인 실행: Cloudflare Cron Triggers

Cloudflare가 정해진 시각에 우리 코드를 자동 호출 → **사람이 버튼 안 눌러도 됨.**

`wrangler.jsonc` 에 추가:
```jsonc
{
  "triggers": {
    "crons": [
      "0 3 * * *"     // 매일 새벽 3시(UTC) 전체 가격 갱신
    ]
  }
}
```

코드 진입점 (`src/index.tsx` 에 `scheduled` 핸들러 추가):
```typescript
export default {
  fetch: app.fetch,
  async scheduled(event, env, ctx) {
    ctx.waitUntil(runPriceCollection(env))  // 무인 수집 실행
  }
}
```

### 수집 부하 관리 (중요)
- 네이버 API는 **하루 25,000회** 호출 제한 (현재 무료 기준)
- 100개 게임 × 하루 1회 = 100회 → 여유 충분
- 게임이 1,000개로 늘면 → 배치 분할(예: 하루 10번에 나눠 100개씩)
- Cron이 한 번에 다 못 돌리면 → "마지막 수집 시각이 오래된 것부터" N개씩 처리

## 6. 관리자가 하는 일 (최소화)

### 6-1. 게임 추가 (제목만)
```
[ + 게임 추가 ]
제목: 젤다의 전설 왕국의 눈물      [등록]
```
→ 등록하면 다음 Cron 때 자동으로 PS5/PS4/스위치... 가격이 채워짐.
→ (선택) "지금 바로 수집" 버튼으로 즉시 1회 수집도 가능.

### 6-2. 레퍼럴 ID 입력 (한 번만)
```
쿠팡 파트너스 ID   : [ AF1234567        ]
G마켓 ESM ID       : [ esm_xxxxx        ]
11번가 제휴 ID     : [ 11st_xxxxx       ]
                                      [저장]
```
→ 저장하면 `/go/:priceId` 리다이렉터가 자동으로 링크에 ID를 붙임.
→ **레퍼럴은 코드/프론트에 절대 노출 안 됨 (서버에서만 결합).**

이 두 가지 외에 관리자가 평소에 할 일은 **없음.**

## 7. 단계별 구현 로드맵

| 단계 | 내용 | 상태 |
|------|------|------|
| 0 | 제목 기반 플랫폼 자동 분류 검증 | ✅ 완료 |
| 1 | `classifyProduct()` 분류 함수 모듈화 (naver.ts 확장) | 다음 |
| 2 | `runPriceCollection(env)` 무인 수집 함수 | 다음 |
| 3 | 에디션 자동 upsert (db.ts) | 다음 |
| 4 | Cron Trigger 등록 + scheduled 핸들러 | 다음 |
| 5 | 검수 큐 (confidence 낮은 항목만) | 추후 |
| 6 | 게임 "제목만" 추가 관리자 UI | 이번 Step1 |
| 7 | 레퍼럴 ID 입력 관리자 UI | 이번 Step1 |

> 이번 Step 1에서는 **6, 7 (가벼운 관리자 UI)** 까지 구현하고,
> 1~4 (무인 엔진)는 함수 골격을 먼저 만들어 둔 뒤 점진 적용합니다.

## 8. 한계와 대비책 (솔직하게)

| 리스크 | 대비책 |
|--------|--------|
| 제목에 플랫폼 표기가 없는 상품 | ETC로 보류 → 검수 큐, 자동 노출 안 함 |
| 중고/리퍼 상품 섞임 (현재 알려진 이슈) | 금지어에 "중고/리퍼/개봉" 추가 |
| 동명이인 게임 (예: 똑같은 제목 다른 게임) | 게임에 출시연도/별칭 필드 추가로 식별 보강 |
| 네이버 API 호출 한도 | 배치 분할 + "오래된 것부터" 갱신 |
| 디지털 가격은 네이버에 안 잡힘 | 추후 Steam/PSN 전용 어댑터 추가 |

---

## 결론

> **관리자는 "게임 제목"과 "레퍼럴 ID"만 넣는다.**
> 나머지(검색→필터→플랫폼분류→에디션생성→가격저장→최저가갱신)는
> Cron이 매일 자동으로 돌린다. 이상한 것만 검수 큐에서 가끔 확인한다.

이 설계가 OK면, 바로 **인라인 아코디언 → 가벼운 관리자 모드** 순서로 구현 들어갑니다.
